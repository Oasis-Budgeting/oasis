## 2024-05-18 - Missing ARIA attributes on Icon-only UI components
**Learning:** React components (especially those wrapping HTML primitives like `<button>`) using just an icon inside lack sufficient context for screen reader accessibility when using CSS visually hidden text is omitted. Icon-only buttons used for application shell navigation (sidebar toggle, theme toggle, and floating action buttons like "Quick Add") are crucial for primary app interaction.
**Action:** Always verify that icon-only buttons (`<Button size="icon">` or native `<button>` tags with just icons inside) have explicit `aria-label` attributes to provide meaning for screen readers. Furthermore, stateful toggles (like sidebar menus) should include `aria-expanded` reflecting their open/closed state, and if there are keyboard shortcut mappings bound to the elements, include `aria-keyshortcuts` to inform assistive technology.
